Network restriction for the jails is **implemented, validated live (Claude passed the Kaimon quiz end-to-end), and merged to master** (the user committed across several sessions). Implements [[project-network-restriction-native-plan]] — no srt, no nested bwraps.

**Architecture:** a network-restricted jail keeps jail.nix's default empty netns (no `network` combinator = kernel deny-all egress) and reaches the internet only through a host-side allowlist HTTP proxy, bridged in over a unix socket by an in-jail `socat`. Claude↔Kaimon MCP is bridged the same way over a shared unix socket.

**Proxy = tinyproxy (switched 2026-07-13, replaced the hand-written Python proxy).** User wanted an established, reviewed tool for the security boundary, not bespoke code. Findings that drove it: home-manager has NO tinyproxy module (verified in the pinned input); the NixOS `services.tinyproxy` is a systemd *system* service (root, one global instance) → unusable for this unprivileged/per-jail/portable design. So we kept the wrapper-launched per-jail model and only swapped the engine. Allowlist = default-deny via `FilterDefaultDeny Yes` + `FilterType ere`; each domain d → anchored `(^|\.)d$` (dots escaped) reproducing the old suffix match; `ConnectPort 443`/`80` limits HTTPS. The security-critical **filter** file is generated at build time (store) by `mkProxyFilterFile name domains` (`writeText`; a filter file is mandatory — tinyproxy has no inline-domains option, and the store copy is safe since the agent can reach neither the store nor the socket dir). `FilterCaseSensitive Off` is correct/safe for an allowlist (case-folding can't turn a denied host into a different allowed one). Validated: allowed hosts+subdomains 200, denied hosts / disallowed CONNECT ports / substring-attack hosts 403.

**Simplified with ip2unix (2026-07-14, replacing the socat/random-port plumbing).** tinyproxy is **TCP-listen only** (no AF_UNIX), which originally forced: a random host port + retry loop, a `/dev/tcp` readiness probe, a runtime-generated conf, and a host-side `socat UNIX-LISTEN → TCP` leg, all split across a `jail-net-proxy` script and `runInner`. Now `runInner` starts `ip2unix -r in,tcp,port=3128,path=$SOCK tinyproxy -d -c <store conf>` directly: ip2unix (nixpkgs; LD_PRELOAD, LGPL, aszlig/nixcloud, NLnet-sponsored — purpose-built for exactly this) makes tinyproxy bind the per-instance unix socket itself; `in`-scoped rule leaves tinyproxy's outgoing connections untouched. Port 3128 is virtual (never bound on host TCP) → no clashes → no retry loop; conf is build-time `mkProxyConf name domains` (`writeText`, references the filter file); socket-file-exists ⟺ tinyproxy bound it (readiness). `jail-net-proxy` deleted; restricted branch of `runInner` is ~15 lines. Caveat: LD_PRELOAD needs dynamically-linked libc binaries (tinyproxy is; would NOT work on Go binaries).

**Instance-safe: several `jailed-claude` or `jailed-julia` can run in parallel** (shared host-home depot/config is normal for Claude/Julia). Proxy is **per wrapper process**, not per jail-name: host socket `= .cache/jail-net/<name>.$$.sock` (keyed by wrapper PID), bound into the jail at the fixed in-jail path via a **runtime** bind — `unsafe-add-raw-args ''--bind "''${JAIL_PROXY_HOST_SOCK-}" /run/jail-net/proxy.sock''` in `restrictedNetOptions` (mirrors how `mount-cwd` binds `$PWD`; the wrapper exports `JAIL_PROXY_HOST_SOCK` before running inner). Per-instance trap tears each proxy down (no shared daemon / refcount / leaks). jail.nix mechanism confirmed by reading its source: combinators can push runtime bwrap args (`add-runtime`/`RUNTIME_ARGS`, `unsafe-add-raw-args` with `$VAR`). **Confirmed live (2026-07-13 socat version: two real `jailed-julia` panes downloaded packages in parallel; re-verified 2026-07-14 ip2unix version: 2 concurrent jails 200/200, allowed 200 / denied 403, no socket or process leaks).** **Kaimon deliberately left single-instance** (one server at a time).

**Layout (all in `nix_src/`) — UPDATED 2026-07-14, see [[project_jailed_agents_devshell]] for the current split:**
`network-restrictions.nix` no longer exists — its content (proxy pkg, ports, socket paths,
cacert, `localhostResolveBinds`, `restrictedNetOptions`) was merged into `jailed-agents.nix`,
which now also owns the generic `makeJailed`/`mkLauncher` (program-agnostic). The Claude↔Kaimon
MCP bridge (`kaimonClientLeg`/`kaimonServerLeg`/`kaimonBridgeBinds`) and the four
`makeJailedClaude/Julia/Kaimon/Shell` constructors moved to `flake.nix` (program-specific).
`share-ns "pid"` was later removed from every jail entirely (see the focused finding in
[[security_review_reports]]) — Kaimon↔Julia comm rides the shared `.cache/kaimon` unix
sockets, no shared PID namespace needed.

**Load-bearing gotchas (hard-won this session, still true):**
- **INVARIANT: never bind `agentshome/.cache/` wholesale (or `.cache/jail-net`) into any jail** — it holds every instance's host-side proxy socket; a jail reaching it could route through another jail's allowlist (e.g. jailed-julia using jailed-claude's domains). Only the single per-instance socket file is bound in. Documented in CLAUDE.md gotchas + a SECURITY comment at `_jn_dir` in `runInner`, and **enforced at eval time** by `assertNoForbiddenBinds` in `jailed-agents.nix` (2026-07-14): jail.nix combinators are opaque `state -> state` lambdas, but the final launcher is a `writeShellApplication` whose `.text` attr exposes the whole bwrap cmd at eval — the check scans it for bind sources equal to a protected path, beneath it, or any ancestor up to `/`. Protected = `.cache/jail-net` (built-in) + `forbiddenBindPaths` from flake.nix (currently `agentshome/.envrc` — agent rewriting it would run on the host if direnv-allowed). Gotchas: `lib.escapeShellArg` leaves safe-char paths UNQUOTED, so match both ` p ` and `'p'` (first version silently never fired); ro binds must be rejected too (unix `connect()` succeeds through a ro mount — sockets are exempt from EROFS); `.git-credentials` deliberately NOT in the list (its ro bind into claude/shell is intended, user confirmed). The check ALSO enforces an allowlist: every static bind source parsed from the bwrap args (`--(ro-|dev-)?bind(-try)?`) must be under agentshome or `builtins.storeDir`, with exactly three exceptions verified against every current inner: `"`-prefixed runtime tokens (`"$PWD"` mount-cwd, `"$(realpath` time-zone, `"$P"`/`"$PATH_TO_BIND"` runtime-deep-ro-bind, the proxy sock var), `~/.local/share/jail.nix/` (jail.nix fake-passwd data in the HOST home), and `/run/systemd/resolve` (jail.nix `network` combinator). `forbiddenBindPaths` currently = agentshome's `.envrc` + `.direnv` (nix-direnv cache — host direnv sources it WITHOUT re-approval, unlike .envrc) + repo-level `.hosthome`, `nix_src`, `.git` — the latter three are redundant with the allowlist today, kept deliberately: the user expects the allowlist may be dropped if it makes new jailed agents harder to add, and wants the denylist self-sufficient. Negative-tested both asserts: `.git-credentials` in the list failed eval on jailed-claude; a `/etc/passwd` ro-bind failed on jailed-shell; drvPath unchanged after (asserts are behavior-free). NOTE: old store generations contain a pre-refactor `jailed-julia-inner` with `--ro-bind agentshome /home/agents` WHOLESALE — a scare during verification; current inners all verified clean, don't trust stale `/nix/store/*-inner` artifacts when auditing.
- **Kaimon wipes `.cache/kaimon/sock/` on startup** → the MCP bridge socket MUST live outside it, in `.cache/kaimon-jail-sock/` (that's why `kaimonBridgeBinds` exists separately from `kaimonCacheWriteBinds`).
- **Empty netns has no `/etc/hosts`/`nsswitch.conf`** → `localhost` won't resolve (EAI_AGAIN), breaking `http://localhost:2828/mcp`; `localhostResolveBinds` provides both (`hosts: files` first) and is now applied automatically to every jail without full host network (not just proxy-restricted ones). Must NOT be combined with the `network` combinator (it binds its own /etc/hosts → conflict) — `makeJailed` asserts `network`/`proxiedNetwork` (renamed from `proxiedNetwork` 2026-07-15) mutually exclusive.
- `getExe`/`getExe'` used instead of hand-written `${pkg}/bin/x` (see [[feedback-nix-idiomatic]]).
- New nix files must be `git add`-ed before `nix eval`/build sees them (flakes only track git-tracked files).
